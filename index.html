Complete Curriculum

1

LLM Behavior + Reliability Foundations

Weeks 1–3

Week 1: LLM Behavior (Now with Realism)

Focus: Tokenization, temperature/sampling, output instability

📚 Learn
  • Tokenization and how models think
  • Sampling, temperature, and why outputs are unstable
  • Why outputs will betray you at 3 AM

Resource: Hugging Face LLM Course - tokenization, transformers basics, generation behavior

🛠️ Build
  • Prompt benchmarking harness across models
  • Log EVERYTHING: input, parameters, output
  • Implement structured logging (JSON format)
🔍 Failure Engineering
  • Log weird outputs and categorize them
  • Track variance across temperature settings
📋 Week 1 Deliverables
  • Logging Harness: Runnable script logging prompts, parameters, outputs
  • Variance Report: Temperature variance analysis with examples
  • Failure Log: Categorized weird outputs with patterns identified

Week 2: Context Engineering (Not Prompt Cosplay)

Focus: System prompts, role separation, context packing

📚 Learn
  • System prompts and their actual impact
  • Role separation and context structure
  • Context packing tradeoffs

Resources: OpenAI Cookbook - prompting patterns, structured outputs, evals; Anthropic Prompting & Context Engineering - system prompts, context design

🛠️ Build
  • Versioned prompt system with test cases per version
  • Regression checks and prompt diff tracking
🔍 Failure Engineering
  • Track prompt regressions—what changed, what broke?
  • Measure performance delta between versions
📋 Week 2 Deliverables
  • Prompt Version History: Git-like tracking with test cases per version
  • Regression Analysis: Before/after performance metrics with delta measurement
  • Context Packing Report: Trade-off analysis between context richness and token efficiency

Week 3: Structured Outputs + Failure Handling

Focus: Schema validation, retry strategies, partial failures

📚 Learn
  • Schema validation and JSON extraction
  • Retry strategies and guardrails
  • Partial failure handling

Resource: OpenAI Cookbook - Structured Outputs

🛠️ Build
  • Strict JSON extraction service with retry logic (exponential backoff)
  • Partial failure handling and invalid output logging dashboard
🔍 Failure Engineering
  • Intentionally break it with malformed inputs
  • Log all failures and recovery attempts
📋 Week 3 Deliverables
  • JSON Extraction Service: Strict schema validation with retry logic implementation
  • Failure Dashboard: Invalid outputs categorized with failure rates and recovery time
  • Guardrails Catalog: Documented retry strategies with exponential backoff curves
2

RAG + Data Pipelines

Weeks 4–9

Week 4: RAG Basics + Ingestion Reality

Focus: Data ingestion, document handling, metadata extraction

📚 Learn
  • Ingestion pipelines and real-world challenges
  • Document formats (PDFs are evil, FYI)
  • Metadata extraction strategies

Resources: LlamaIndex Docs - ingestion, chunking, pipelines; Designing Data-Intensive Applications - data modeling, pipelines, system thinking

🛠️ Build
  • Ingestion pipeline handling duplicate detection and removal
  • Corrupted file recovery and missing metadata logging
📊 Data Quality
  • Data cleaning pipeline and quality metrics for ingested data
📋 Week 4 Deliverables
  • Ingestion Pipeline: End-to-end document processing with duplicate detection
  • Data Quality Report: Ingestion metrics showing success rate, error types, recovery actions
  • Metadata Extraction Log: Successfully extracted vs missing metadata with patterns

Week 5: Chunking + Data Quality

Focus: Chunking strategies, semantic fragmentation, dataset versioning

📚 Learn
  • Chunking strategies and tradeoffs
  • Semantic fragmentation and context windows
  • Dataset versioning and tagging approaches
  • Overlap strategies and their impact on retrieval

Resource: LlamaIndex Docs - Chunking Strategies - chunking optimization, experimentation

🛠️ Build
  • Chunking experiments with multiple strategies
  • Track chunk overlap issues and semantic fragmentation
  • Implement dataset versioning (simple tags work)
📊 Metrics
  • Chunk quality evaluation and coverage analysis
📋 Week 5 Deliverables
  • Chunking Analysis: Multiple strategy comparison (fixed size, semantic, sliding window) with overlap metrics
  • Dataset Versioning System: Git-like tags with changelog and rollback capability
  • Fragmentation Report: Semantic coherence analysis showing optimal chunk boundaries

Week 6: Hybrid Retrieval

Focus: BM25 + vector search, metadata filtering, schema design

📚 Learn
  • BM25 keyword-based search fundamentals
  • Vector search and semantic similarity
  • Combining hybrid approaches for better recall
  • Metadata filtering strategies and schema design

Resources: Pinecone RAG Series - retrieval strategies, hybrid search; Weaviate Docs - hybrid search, filtering, schema design

🛠️ Build
  • BM25 + Vector search integration
  • Metadata filtering: time-based, source, type-based
  • Schema design that actually scales
🔍 Failure Engineering
  • Test edge cases with malformed metadata
  • Measure query coverage with hybrid vs single approach
📋 Week 6 Deliverables
  • Hybrid Retrieval System: BM25 + vector search integration with metadata filtering
  • Query Coverage Analysis: Comparison table showing recall improvement with hybrid approach
  • Schema Design Docs: Database schema with indexing strategy and query performance metrics

Week 7: Reranking

Focus: Reranker integration, quality comparison, latency analysis

📚 Learn
  • How rerankers work: semantic vs BM25 quality
  • Reranker models and their tradeoffs
  • Cost vs quality: when reranking pays off
  • Latency impact analysis for production

Resource: Pinecone Reranking Guide - reranking strategies, quality metrics

🛠️ Build
  • Reranker integration
  • Comparison analysis: before/after reranking quality, latency impact
🔍 Failure Engineering
  • Intentionally inject irrelevant docs
  • Measure reranker robustness
📋 Week 7 Deliverables
  • Reranker Comparison: Before/after quality metrics showing precision and MRR improvement
  • Latency Impact Analysis: Trade-off graph between quality gains and latency cost
  • Robustness Test Report: Reranker performance against adversarial/irrelevant document injections

Week 8: Retrieval Evaluation (Not Fake Metrics)

Focus: Evaluation datasets, precision/recall, real metrics

📚 Learn
  • Evaluation dataset creation best practices
  • Precision@k, Recall@k, and MRR metrics
  • Human vs automatic evaluation disagreement
  • Real-world evaluation challenges

Resource: LlamaIndex Evaluation Docs - evaluation frameworks, metrics

🛠️ Build
  • Evaluation dataset (50+ queries minimum)
  • Real metrics: Precision@k, Recall@k, MRR (Mean Reciprocal Rank)
  • Compare human labels vs auto labels and measure disagreement
📋 Week 8 Deliverables
  • Evaluation Dataset: 50+ queries with human-labeled ground truth and relevance judgments
  • Metric Comparison Report: Human vs automatic evaluation agreement analysis with correlation metrics
  • Benchmark Baseline: Precision@5, Recall@10, MRR scores with expected thresholds

Week 9: Trustworthy Answers

Focus: Citation systems, abstention logic, confidence scoring

🛠️ Build
  • Citation system (track sources) and abstention logic (say "I don't know")
  • Confidence scoring system

Resource: Anthropic Context Engineering - citation strategies, confidence scoring

🔍 Failure Engineering
  • Force hallucination scenarios and measure how often system refuses to answer
📋 Week 9 Deliverables
  • Citation System: Source tracking with citation accuracy and hallucination detection
  • Confidence Scoring Implementation: Calibrated confidence metrics with accuracy curves
  • Abstention Analysis: Frequency analysis of system refusals showing precision of refusal logic
3

LLMOps + Evals + Cost + Latency

Weeks 10–15

Week 10: Observability

Focus: Tracing infrastructure, structured logging, latency tracking

🛠️ Build
  • Tracing infrastructure (LangSmith or Phoenix)
  • Structured logs: prompt (versioned), retrieval results, LLM output, latency at each stage

Resources: LangSmith Docs - tracing, evaluation workflows; Arize Phoenix - observability, debugging

🔍 Failure Engineering
  • Replay failures for debugging and build dashboards showing failure patterns
📋 Week 10 Deliverables
  • Tracing Dashboard: LangSmith/Phoenix integration showing end-to-end request traces
  • Structured Logging System: Formatted logs with prompt version, retrieval results, latency breakdown
  • Failure Replay Toolkit: System to replay failed requests with full context for debugging

Week 11: Evaluation Systems (Not Just Tools)

Focus: Eval philosophy, LLM-as-judge, accuracy measurement

📚 Learn
  • Eval philosophy—what should you measure?
  • When LLM evals fail

Resource: OpenAI Evals Cookbook - eval design, LLM-as-judge

🛠️ Build
  • LLM-as-judge evaluation system
  • Compare LLM judgment to human judgment: agreement, disagreement, eval accuracy
📋 Week 11 Deliverables
  • LLM-as-Judge Framework: Prompt design and validation scoring rubric
  • Judge Calibration Report: Human vs LLM judge agreement analysis with disagreement reasons
  • Eval Philosophy Doc: Documented approach to what should be measured and why

Week 12: Experimentation

Focus: A/B testing, regression detection, statistical significance

📚 Learn
  • A/B testing fundamentals and experimental design
  • Statistical significance and p-values
  • Regression detection strategies
  • Baseline establishment and drift detection
🛠️ Build
  • A/B testing system with baseline metrics
  • Regression detection pipeline and statistical significance testing
🔍 Failure Engineering
  • Deliberately introduce regressions
  • Verify system detects them
📋 Week 12 Deliverables
  • A/B Testing System: Baseline metrics with statistical significance testing framework
  • Regression Detection Pipeline: Automated alerts when metrics fall below threshold
  • Experiment Report: p-values, confidence intervals, and power analysis documentation

Week 13: Cost Engineering (Money Exists)

Focus: Token tracking, cost metrics, quality tradeoffs

🛠️ Build
  • Token tracking dashboard with cost per request and per user metrics
  • Cost vs quality tradeoff analysis

Resource: OpenAI Production Best Practices - cost optimization, production patterns

📋 Week 13 Deliverables
  • Token Tracking Dashboard: Cost per request, per user, per model with trends over time
  • Cost Optimization Report: Model selection analysis showing cost vs quality trade-offs
  • Budget Forecast: Projected monthly costs with scaling scenarios

Week 14: Latency Engineering

Focus: Async handling, streaming responses, percentile metrics

🛠️ Build
  • Async request handling and streaming responses
  • Latency metrics: P50 (median), P95 (tail), P99 (extreme tail)

Resource: Latency Optimization Guide - optimization strategies, latency reduction

🔍 Failure Engineering
  • Simulate slow APIs and test timeout handling
📋 Week 14 Deliverables
  • Async Request Handler: Queue-based request processing with priority levels
  • Latency Metrics Dashboard: P50, P95, P99 percentiles with tail latency analysis
  • Timeout Resilience Report: Graceful degradation test results under slow API conditions

Week 15: Model Routing

Focus: Routing logic, cost constraints, complexity-based routing

📚 Learn
  • Routing logic strategies and patterns
  • Cost-aware routing and optimization
  • Query complexity detection methods
  • Confidence-based routing decisions
🛠️ Build
  • Routing system based on query complexity, confidence scores, cost constraints
  • Route to different models/APIs strategically
🔍 Failure Engineering
  • Test routing with malformed requests
  • Measure fallback behavior when preferred model is unavailable
📋 Week 15 Deliverables
  • Routing System: Logic-based routing (complexity/cost/confidence) with fallback chains
  • Model Performance Comparison: Quality metrics by model with optimal routing thresholds
  • Fallback Test Report: System behavior when primary model unavailable or degraded

Agents + Control + HITL

Weeks 16–19

Week 16: Tool Use + Failure Handling

Focus: Tool calling, failure recovery, idempotency

📚 Learn
  • Tool calling infrastructure and protocols
  • Error handling and recovery strategies
  • Idempotency and state management
  • Tool validation and security constraints

Resource: OpenAI Agents / Tools Docs - tool implementation, error handling

🛠️ Build
  • Tool calling infrastructure
  • Tool failure handling: retry logic, fallback strategies, idempotency checks
  • Tool logging and monitoring
🔍 Failure Engineering
  • Force tool timeouts and test recovery
  • Test duplicate tool calls and idempotency
📋 Week 16 Deliverables
  • Tool Calling Infrastructure: Schema definition with automatic validation and error handling
  • Tool Execution Log: Complete trace of tool calls with failures, retries, and idempotency checks
  • Recovery Strategy Documentation: Fallback logic and retry patterns with test results

Week 17: Stateful Orchestration

Focus: State management, checkpointing, recovery

📚 Learn
  • Workflow state management patterns
  • Checkpointing and persistence strategies
  • Recovery and replay mechanisms
  • State consistency and distributed concerns

Resource: LangGraph - stateful workflows, orchestration

🛠️ Build
  • Workflow state management with checkpointing system
  • Recovery from crashes: resume from checkpoint, replay logic, state consistency checks
🔍 Failure Engineering
  • Simulate crashes at different workflow stages
  • Test recovery and state consistency
📋 Week 17 Deliverables
  • Workflow Orchestration System: LangGraph-based stateful workflow with checkpoint persistence
  • Crash Recovery Test Suite: Simulated failures at each stage with recovery verification
  • State Consistency Report: Analysis showing no data loss or corruption on recovery

Week 18: Multi-Agent (Skeptically)

Focus: Multi-agent patterns, comparative analysis

📚 Learn
  • Multi-agent patterns and pitfalls
  • When multi-agent is actually useful (hint: rarely)
🛠️ Build
  • Multi-agent experiment—CRITICALLY measure if it's actually better
  • Single agent vs multi-agent quality, cost, and latency comparison

Resource: Microsoft AutoGen - multi-agent experiments

📋 Week 18 Deliverables
  • Multi-Agent Implementation: Team of specialized agents with communication protocol
  • Comparative Analysis: Single-agent vs multi-agent quality, cost, and latency benchmark
  • Honest Assessment Memo: Was multi-agent actually better? Recommendation with reasoning

Week 19: Human-in-the-Loop

Focus: Approval systems, audit logging, rollback capability

📚 Learn
  • Human-in-the-loop workflow design patterns
  • Audit logging and compliance requirements
  • Decision tracking and rollback strategies
  • User interface considerations for approvals
🛠️ Build
  • Approval system for high-stakes decisions
  • Audit logging (track all decisions)
  • Rollback capability: revert decisions and replay with human override
🔍 Failure Engineering
  • Test audit log integrity under concurrent operations
  • Verify rollback consistency across system
📋 Week 19 Deliverables
  • HITL Approval Workflow: UI and backend for human decision points with timeout handling
  • Audit Log System: Immutable transaction log showing all approvals, rejections, and overrides
  • Rollback Capability Demo: Proof of ability to revert decisions and replay with human input

Fine-Tuning

Weeks 20–21

Week 20: PEFT + Dataset Reality

Focus: Parameter-efficient fine-tuning, dataset creation, problem analysis

📚 Learn
  • Parameter-Efficient Fine-Tuning (PEFT), LoRA, QLoRA, and alternatives
  • When fine-tuning actually helps

Resources: Hugging Face PEFT - parameter-efficient fine-tuning; Full Stack Deep Learning - when NOT to fine-tune

🛠️ Build
  • Small fine-tuning experiment with dataset creation pipeline
  • Data collection, cleaning, and quality checks
📋 Week 20 Deliverables
  • Fine-Tuning Experiment: LoRA implementation with dataset creation and quality assurance
  • Fine-Tune Performance Report: Quality metrics showing improvement on target domain
  • When-to-FT Decision Framework: Analysis of problem characteristics determining fine-tuning viability

Week 21: RAG vs Fine-Tuning (Real Comparison)

Focus: Comparative analysis, tradeoff evaluation

📚 Learn
  • RAG vs fine-tuning tradeoffs
  • Quality metrics comparison methodology
  • Cost-benefit analysis frameworks
  • Maintenance burden considerations
🛠️ Build
  • Intentionally build a bad fine-tune and compare RAG vs fine-tuned model
  • Compare on: quality metrics, latency, cost, maintainability
📝 Deliverable
  • Tradeoff memo: latency comparison, cost analysis, maintenance burden, recommendation with reasoning
📋 Week 21 Deliverables
  • RAG vs Fine-Tuning Benchmark: Parallel systems showing quality, latency, and cost metrics
  • Decision Matrix Memo: Detailed analysis of trade-offs with pros/cons and recommendation
  • Maintenance Analysis: Long-term operational burden comparison with update frequency and rollback complexity

Week 24: Scaling + Production Resilience (CRITICAL)

  • Resilience Test Results: Chaos engineering experiments with service recovery times and SLA compliance
